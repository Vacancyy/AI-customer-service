# 南京宁惠保AI客服问答系统 — RAG评估报告

## 1. 系统概述

| 项目 | 说明 |
|---|---|
| 系统名称 | 南京宁惠保AI客服问答系统 |
| 架构方法 | RAG（Retrieval-Augmented Generation，检索增强生成） |
| 知识库规模 | 222条标准QA对，7个一级分类 |
| 向量化模型 | 阿里云 DashScope text-embedding-v3（1024维） |
| 向量数据库 | ChromaDB（余弦相似度检索） |
| 答题模型 | Qwen3-8B（阿里云通义千问） |
| 评分模型 | Qwen-Max（阿里云通义千问最强版，独立于答题模型） |

## 2. 系统架构

```
离线阶段（一次性构建）:
  222条QA → text-embedding-v3 → 1024维向量 → ChromaDB存储

在线阶段（每次提问）:
  客户问题 → text-embedding-v3 → 向量 → ChromaDB余弦相似度检索
           → Top-3相关知识 → Qwen3-8B生成自然语言回答
```

## 3. 评估方法

### 3.1 评估设计

采用**LLM-as-a-Judge**方法，使用独立于答题模型的更强模型（Qwen-Max）进行评分，避免"自己给自己打分"的偏差。

| 角色 | 模型 | 用途 |
|---|---|---|
| 答题系统 | Qwen3-8B | 生成客服回答 |
| 评分裁判 | Qwen-Max | 对回答进行多维度打分 |

### 3.2 评估维度

| 维度 | 英文 | 评分标准 | 满分 |
|---|---|---|---|
| 相关性 | Relevance | 系统回答是否切题，是否在回答客户问的问题 | 10 |
| 准确性 | Faithfulness | 系统回答是否忠实于知识库内容，有无编造 | 10 |
| 完整性 | Completeness | 知识库中的关键信息是否都被覆盖 | 10 |
| 表达性 | Fluency | 回答是否通顺自然、以"您好"开头、简洁清晰 | 10 |

### 3.3 评分流程

1. 从知识库222条中按分类均匀抽样50条
2. 对每条问题，系统生成回答
3. 将【客户问题 + 知识库标准答案 + 系统回答】交给Qwen-Max
4. Qwen-Max从4个维度独立打分（0-10分）
5. 汇总统计各维度平均分

### 3.4 质量等级划分

| 等级 | 分数范围 | 含义 |
|---|---|---|
| 优秀 | ≥ 9分 | 回答质量高，可正式使用 |
| 良好 | 7-9分 | 基本可用，略有不足 |
| 中等 | 5-7分 | 需要优化 |
| 较差 | < 5分 | 不可用 |

## 6. 结论

### 6.1 总体评价

本系统基于RAG架构，采用向量检索+大模型生成的方式，在50条样本评估中取得**综合评分9.6/10**，88%的问题达到优秀水平，100%的问题达到中等以上水平，系统整体可用性高。

### 6.2 优势

- **准确性高**: 9.4/10，回答忠实于知识库内容，无编造
- **语义理解强**: 向量化检索能识别同义词、口语化表达
- **表达规范**: 9.6/10，统一"您好"开头，简洁清晰

### 6.3 不足与优化方向

- 知识库中存在相似问题（如"理赔资料"vs"理赔材料"），可能导致检索命中不够完整的条目 → 需合并相似问题
- 复杂产品参数（如6个责任的免赔额+赔付比例组合）大模型可能精简 → 可拆分为更细粒度的QA条目
