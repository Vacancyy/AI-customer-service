# 客服智能问答系统 — 工作进度汇报

**汇报人**: 谢程
**日期**: 2026年5月6日

## 一、系统定位

基于南京宁惠保222条标准QA知识库，构建AI智能客服问答系统。客户用自然语言提问，系统自动检索知识并生成准确回答。

---

## 二、技术方案

### 2.1 架构：RAG（检索增强生成）

```
客户提问
   ↓
① 向量检索：text-embedding-v3 将问题转为1024维向量 → ChromaDB余弦相似度检索Top-3
   ↓
② 大模型生成：将检索到的知识 + 客户问题交给 Qwen3-8B → 生成自然语言回答
   ↓
客服回复
```

为什么不选微调训练而选RAG：
- 训练需要大量标注数据，成本高、周期长
- 知识更新需要重新训练，RAG只需更新知识库即可生效
- RAG回答有据可查，不会凭空编造

### 2.2 为什么不加Rerank

已实测对比，当前222条知识库Rerank收益极小：

| 指标 | 只有Embedding | Embedding + Rerank |
|---|---|---|
| 综合评分 | 9.6 | 9.6 |
| 优秀数/50条 | 44 | 45 |
| 中等数/50条 | 1 | 0 |
| API调用次数/每次提问 | 2次 | 3次（多一次Rerank） |
| 口语化问题通过率 | 直通过 | Rerank分数低差点误杀 |

结论：222条数据量下Embedding一次就能精准检索，Rerank多花1次API调用但分数没提升。当知识库扩展到数千条时再考虑加入Rerank精排。

### 2.3 技术组件

| 组件 | 选型 | 作用 |
|---|---|---|
| 向量生成 | 阿里云 text-embedding-v3 | 文本 → 1024维向量 |
| 向量存储 | ChromaDB | 本地向量数据库，余弦相似度检索 |
| 回答生成 | 阿里云 Qwen3-8B | 根据检索结果生成自然语言回答 |
| 知识库 | 222条标准QA（7个分类） | 回答的知识来源 |

## 三、已完成工作

### 3.1 向量化知识库构建

222条QA通过Embedding API转为1024维向量，存入ChromaDB。首次构建后持久化存储，重启无需重建。

### 3.2 语义理解能力

**同义词识别**

| 客户说的 | 系统理解的 |
|---|---|
| 门槛费 / 起付线 | 免赔额 |
| 报销 / 赔付 | 理赔 |
| 材料 / 文件 / 东西 | 资料 |
| 取消保险 / 撤单 | 退保 |
| 多少钱 / 收费 | 保费 |

**口语化理解**

| 口语提问 | 系统理解 |
|---|---|
| 自己掏多少才开始赔 | 免赔额是多少 |
| 住院了怎么拿钱 | 如何申请理赔 |
| 买后悔了能退吗 | 怎么退保 |
| 这个保险干嘛用的 | 南京宁惠保是什么 |
| 怀孕生孩子能赔吗 | 哪些情况不赔付 |

**不同语法表达**

| 语法变体 | 系统理解 |
|---|---|
| 我不是南京人能买吗 | 投保条件 |
| 外地户口可以买吗 | 投保条件 |
| 什么人可以投保 | 投保条件 |
| 谁能买这个保险 | 投保条件 |

### 3.3 回答生成规范

- 以"您好"开头，符合客服话术规范
- 关键数字完整输出（免赔额1.5万/2万、保费99元/150元、赔付比例90%/60%等）
- 区分产品版本（基础版99元 vs 升级版150元）
- 知识库没有的内容不编造，引导至人工客服

## 四、评估结果

### 4.1 评估方法

采用 **LLM-as-a-Judge**：用独立于答题模型的更强模型担任裁判打分，避免"自己评自己"。

| 角色 | 模型 | 说明 |
|---|---|---|
| 答题系统 | Qwen3-8B | 生成客服回答 |
| 评分裁判 | Qwen-Max | 独立打分，更严格客观 |

评估流程：从222条中均匀抽样50条 → 系统回答 → 交给Qwen-Max四维度打分

### 4.2 四维度评分

| 维度 | 含义 | 得分 / 满分 |
|---|---|---|
| 相关性 (Relevance) | 回答是否切题 | **9.8 / 10** |
| 准确性 (Faithfulness) | 是否忠实于知识库，无编造 | **9.4 / 10** |
| 完整性 (Completeness) | 关键信息是否覆盖 | **9.7 / 10** |
| 表达性 (Fluency) | 是否通顺规范 | **9.5 / 10** |
| **综合评分** | | **9.6 / 10** |

### 4.3 质量分布

| 等级 | 数量 | 占比 |
|---|---|---|
| 优秀（≥9分） | 45 | 90% |
| 良好（7-9分） | 5 | 10% |
| 中等（5-7分） | 0 | 0% |
| 较差（<5分） | 0 | 0% |

**100%达到良好以上，90%达到优秀，无中等及以下回答。**

### 4.4 典型回答示例

```
客户: 门槛费是多少
系统: 您好，门槛费（即免赔额）根据责任不同有所差异。
      责任一基础款超过1.5万元、升级版超过1.45万元可申请理赔；
      责任二、三基础款共计超过2万元、升级版超过1.95万元可申请理赔；
      责任四、五无免赔额；责任六超过1000元可申请理赔。
```

```
客户: 买后悔了能退吗
系统: 您好，如果您购买"南京宁惠保"后买后悔了，是否可以退保需根据投保时间判断：
      1. 在保单生效前（2026年1月1日前），可以申请撤单或退保；
      2. 保障生效后无法申请退保，特殊情况请联系客服4000040181。
```

## 五、当前局限

| 问题 | 原因 | 影响 |
|---|---|---|
| 个别相似问题检索不够精准 | 知识库中"理赔资料"和"理赔材料"为两个条目 | 偶尔命中不够完整的条目 |
| 复杂数据大模型可能精简 | 6个责任的免赔额+比例组合信息量大 | 细节略有丢失 |
| 单轮问答 | 无上下文记忆 | 追问"那升级版呢"无法理解 |

## 七、系统运行方式

```bash
# 启动客服系统
python3 scripts/ai_qa_system_v2.py

# 重建向量库（知识库更新后）
python3 scripts/ai_qa_system_v2.py --rebuild

# 运行RAG评估
python3 scripts/rag_eval.py 50
```

相关文件：
- 问答系统: `scripts/ai_qa_system_v2.py`
- 评估工具: `scripts/rag_eval.py`
- 知识库: `05_analyze/reports/知识库_优化版.json`
- 向量数据库: `06_models/chroma_db/`
- 评估数据: `05_analyze/reports/rag_eval_report.json`
