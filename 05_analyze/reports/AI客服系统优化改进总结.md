# AI客服系统优化改进总结

## 背景

基于RAGAS + DeepEval综合评估报告，AI客服系统存在5个需改进的问题：

| 优先级 | 问题 | 当前指标 |
|--------|------|----------|
| 高 | 回答冗长 | 简洁度0.75，相关性0.82 |
| 高 | 复杂问题遗漏细节 | 事实正确性0.87 |
| 中 | fallback回答矛盾 | 知识库无答案时先拼凑后转人工 |
| 中 | 知识库覆盖不足 | "门诊费用报销"检索精度0.00 |
| 低 | PII防护可增强 | 平均分0.93 |

---

## 改动详情

### 改进1：Prompt优化 —— 高优先级

**文件**：`scripts/ai_qa_system_v2.py` 第228-271行

**改动**：将原prompt中的长段"要求"列表重构为结构化prompt

**新增内容**：
- 【回答原则】区块：约束只回答用户问题、简洁回答3-5句、优先最直接相关信息
- 【回答风格示例】：给出好坏对比示例
  - ❌ "犹豫期多久" → 展开一堆无关内容
  - ✓ "犹豫期多久" → "您好，这款产品无犹豫期"
  - ❌ "保费多少钱" → 自动扩展双通道报销等
  - ✓ "保费多少钱" → "您好，基础版保费99元，升级版保费150元"

**删除内容**：
- 原要求中过度引导展开的条目（"一字不差地保留"、"综合回答"、"有条理适当分段"）

**代码结构**：拆分出 `_build_prompt()` 和 `_build_context()` 两个独立方法

---

### 改进2：max_tokens降低 —— 高优先级

**文件**：`scripts/ai_qa_system_v2.py` 第210行

**改动**：`max_tokens: 600` → `max_tokens: 400`

**目的**：限制AI输出长度，配合prompt中的"3-5句话"约束，减少冗长回答

---

### 改进3：Fallback逻辑优化 —— 中优先级

**文件**：`scripts/ai_qa_system_v2.py` 第273-301行（`get_answer`方法）

**改动**：

| 项目 | 改动前 | 改动后 |
|------|--------|--------|
| 最低阈值 | 0.3 | 0.5 |
| 分层逻辑 | 二元判断（有/无匹配） | 三级置信度分层 |
| 转人工提示 | 固定在末尾硬加 | 仅中置信度时补充 |

**三级置信度分层**：
- **≥0.7（高置信度）**：直接回答，不附加任何提示
- **0.5-0.7（中置信度）**：回答 + 补充"如需进一步帮助，请咨询人工客服：4000040181"
- **<0.5（低置信度）**：直接返回"这个问题需要咨询人工客服获取准确解答"

**解决的问题**：原逻辑在阈值0.3时检索到不相关内容，AI先拼凑矛盾答案再加转人工，现在低质量检索直接跳过

---

### 改进4：知识库补充 —— 中优先级

**文件**：`05_analyze/reports/知识库_优化版.json`

**改动1：发票丢失QA补救措施**

改动前（条款解释分类）：
> 您好，医疗费用发票原件是保险公司核实您实际发生的医疗费用的主要凭证，若医院开具的发票原件丢失就无法正常为您办理理赔了...

改动后：
> 您好，若您的医疗费用发票原件丢失，建议您拨打宁惠保客服热线4000040181，由人工客服为您确认具体的处理方式。请务必妥善保管发票原件，以免影响理赔。

新增补充条目（理赔材料/发票问题分类，source=supplemented），answer同上，确保从"理赔材料"角度也能检索到

**改动2：门诊费用报销QA合并**

改动前（analysis条目）：
> 宁惠保理赔范围包括住院和门特、门诊大病类型的门诊费用，经医保结算之后，剩余的费用达到免赔额的可以申请理赔...

改动后：合并excel条目中的完整分类列表，answer包含：
> 门诊特殊病、门诊大病、通过"双通道"在门诊（含药店）使用国家谈判药品（升级版）、门诊日间手术（升级版）...普通门诊和门诊急诊不在保障范围内...

**改动3：直赔和快赔QA重新定义**

改动前：只描述"快速理赔"和"传统理赔"，未提及"直赔"

改动后：用①②③逐条列出三种理赔方式：
> ①直赔：42家直赔医院出院即赔，无需提交材料
> ②快赔：仅需上传身份证明和银行信息
> ③传统理赔：需上传全部材料

keywords也从`理赔,保险,材料,费用,申请`更新为`直赔,快赔,理赔,区别,材料,申请`

**总条数**：222 → 223（新增1条supplemented条目）

---

### 改进5：PII输入过滤 —— 低优先级

**文件**：`scripts/ai_qa_system_v2.py` 第13行、第98-102行、第276行

**改动**：
- 新增 `import re`
- 新增 `filter_pii(text)` 函数，脱敏身份证号（18位）和手机号（11位）
- 在 `get_answer()` 入口处调用 `question = filter_pii(question)` 进行脱敏

**脱敏规则**：
- `\d{17}[\dXx]` → `[身份证号已脱敏]`
- `1[3-9]\d{9}` → `[手机号已脱敏]`

## 验证方法

1. **重建向量库**（知识库条数变更后必须执行）：
   ```bash
   python scripts/ai_qa_system_v2.py --rebuild
   ```

2. **手动测试关键问题**：
   - "犹豫期多久" → 应简洁回答"无犹豫期"，不展开等待期/保障期
   - "发票丢失怎么办" → 应直接转人工或给出明确指引
   - "住院手术费用能否报销" → 应列出基础版和升级版的区别
   - "什么是直赔" → 应明确区分直赔/快赔/传统理赔

3. **运行评估脚本**：
   ```bash
   python scripts/ragas_eval.py --sample_size 20
   python scripts/deepeval_eval.py --sample_size 12
   ```
