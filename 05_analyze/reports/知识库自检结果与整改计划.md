## 一、最严重的问题：期数标注不清

**当前知识库大量回答引用的是"六期"产品信息，但产品已进入七期。**

涉及问题的条目（含"六期""2026年""26年"等字样）：

| 条目 | 问题 | 处理方式 |
|---|---|---|
| 已经划扣，需要取消，如何操作退费? | 写了"2026公众号年1月1日前"（明显乱码） | 修正日期，加注"以上期信息为参考" |
| 宁惠保六期理赔时间是什么时候？ | 写"26年理赔通道开启后申请" | 需确认七期理赔时间 |
| 如何申请宁惠保理赔？ | 写"宁惠保六期理赔需等待2026年理赔通道开启" | 需确认七期理赔时间 |
| 理赔标准是什么？ | 写"2026年免赔额为..." | 需确认七期免赔额是否变化 |
| 省外医保人员是否可以参保 | 写"2026年1月1日前居住证明" | 需确认七期要求 |
| 宁惠保是否可以报销住院费用？ | 写"2025年宁惠保设有两个免赔额" | 明确引用旧期数，需更新 |
| 理赔未通过的原因是什么？ | 写"25年入院，请去五期申请理赔" | 已过时，需更新 |
| 理赔申请被驳回的原因是什么？ | 写"25年的特药..." | 已过时，需更新 |
| 如何处理已投保但单位已为其办理团险的情况？ | 写"退款登记截止时间为1月19日12:00" | 已过期 |
| 去年已购买了"南京宁惠保"，今年还需要重新购买吗？ | 写"六期，保障时间是2026年" | 需更新为七期信息 |
| 投保后，为什么一直显示出单中？ | 写"保障是2026年1月1日才开始" | 需确认七期生效时间 |
| 这款产品什么时候生效？ | 写"保障是从2026年1月1日开始" | 需确认七期生效时间 |
| 这款产品保障多长时间？ | 写"2026年1月1日至2026年12月31日" | 需确认七期保障期 |
| 参保有截止日期吗？ | 写"即日起到2025年12月31日" | 已过期 |
| 如何申请南京宁惠保医疗理赔？ | 写"2025年保单出单期为2月20日" | 已过时 |
| 今年还支持直赔吗？直赔医院有哪些？ | 写"42家直赔医院" | 需确认七期直赔医院名单 |

**处理原则：**
- 能确认七期信息的 → 直接更新
- 无法确认的 → 加注"以上一期信息为参考，具体以七期条款为准"

## 四、回答不直接（40+条）

LLM审查标记了大量"不切题"的问题，大部分是回答绕弯子、不直接给结论。但这些问题AI客服实际使用时，大模型会根据知识库内容重新组织语言，**实际影响不大**。

真正需要人工改进的（回答严重不直接的）：

| 条目 | 问题 |
|---|---|
| 如何办理退保手续（无问号） | 回答的是"重复短信处理"，不是退保流程 |
| 免赔额具体金额是多少？ | 回答给的是赔付比例表，没直接说免赔额 |
| 什么是免赔额？免赔额是多少？ | 解释了免赔额概念，但没说具体金额 |

### 6.2 期数来源标注（28条）

所有引用六期/2026年信息的回答末尾统一加注：
```
（注：以上信息为六期产品信息，仅供参考。具体以七期条款为准。）
```

同时修复了"2026公众号年1月1日前"的乱码，改为"保单生效前（六期2026年1月1日前）"。

### 6.3 时期标签添加（178条）

每条QA新增 `period` 字段，按问题内容自动分类：
- 销售期: 50条（涉及参保、投保、购买等）
- 理赔期: 56条（涉及理赔、赔付、报销等）
- 通用: 72条（同时涉及或都不涉及）
